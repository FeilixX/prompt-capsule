## Problem Statement

Prompt Capsule started as a narrow utility: 小红书正文和图片不适合承载长 prompt,读者复制困难,OCR 会错,创作者需要一个短期 `text/plain` URL。

That problem is real, but the bigger product is not "pastebin for prompts." The better product is an on-ramp into agent-native content: a place where AI 小白使用一两次后意识到 **URL can be an executable instruction packet for an agent**.

The product should teach this by doing, not by explaining. A user opens one or two curated capsules, gives the URL to Codex/Claude/Cursor, sees the agent read the URL and complete a task, then understands why they might create their own capsule.

## What Makes This Cool

The aha moment:

```text
给 Codex / Claude:
读取 n78.xyz/c/xxxx, 然后按里面的步骤执行。
```

The user sees the agent fetch a plain-text URL, read an instruction packet, and produce a result. That reframes URL from "webpage link" into "agent-readable content object."

This creates a new content primitive:

- A social post can carry a tiny URL instead of a folded prompt.
- An agent can consume the URL directly.
- A creator can package a repeatable agent workflow as a capsule.
- A community can form around curated, executable instruction packets, not screenshots of prompts.

The point is not prompt collecting. The point is agent literacy through a concrete artifact.

## Constraints

- First public version must be 95-point, not MVP. Development cost is low enough that half-built is the wrong tradeoff.
- Ship Web + MCP + skill together. The concept breaks if it is only a webpage.
- Keep `/c/{slug}` pure `text/plain; charset=utf-8`.
- Do not claim privacy or encryption. Say "knowing the link can read it."
- Public discoverable content is curated. No UGC firehose in v1.
- Safety gates are day-one: prompt injection, dangerous shell, secrets, porn/solicitation, contact/QR/link farming.
- The first user is an AI 小白 who just learned the concept of agent, not an expert prompt engineer.
- Visual game-feel can exist, but the product cannot feel like gambling, cheap gacha, or mecha dashboard.

## Landscape Context

The outside world already has prompt marketplaces and prompt communities. PromptBase/FlowGPT-like products make prompt content browseable, sellable, or remixable. That is not the wedge.

MCP already formalizes `Prompts`, `Resources`, and `Tools` as server capabilities. The official spec treats prompts as reusable templates, resources as URI-addressed context, and tools as callable actions.

Sources:

- MCP Prompts: https://modelcontextprotocol.io/specification/2025-06-18/server/prompts
- MCP Resources: https://modelcontextprotocol.io/specification/2025-06-18/server/resources
- MCP Tools: https://modelcontextprotocol.io/specification/2025-06-18/server/tools
- MCP security discussion is active because tool poisoning and prompt injection are real risks: https://arxiv.org/abs/2603.22489 and https://arxiv.org/abs/2601.17549
- FlowGPT-style prompt communities show demand for sharing prompts, but also show why open UGC needs curation and moderation: https://arxiv.org/abs/2408.00512 and https://arxiv.org/abs/2601.14324

Eureka: existing products treat prompts as templates, marketplace items, or chatbot personas. Prompt Capsule should treat a URL as an agent-native content packet that can travel through social media and be consumed by an agent.

## Premises

1. The first user is a large pool of AI 小白 who just have the concept of agent.
2. The first win is not "I found a good prompt." It is "the agent read a URL and did the thing."
3. The product is agent-native content literacy. UGC comes after the user learns the pattern.
4. Web + MCP + skill must be present in the first serious release.
5. The curated public pool is not raw UGC. Early public capsules are editor-picked.
6. `/c/{slug}` is the protocol core and must never be polluted by UI.
7. The strongest acquisition wedge is the existing 小红书 traffic boost around AI/Codex/vibe coding content.

## Approaches Considered

### Approach A: Capsule Utility + Showcase

Summary: `/new` is the main product. The homepage showcases curated capsules, but creation is the obvious first job.

Effort: M
Risk: Low

Pros:
- Fastest path to a trustworthy utility.
- Solves the original 小红书 copy/OCR pain directly.
- Easy to explain and easy to verify technically.

Cons:
- Too close to pastebin for prompts.
- Does not strongly teach "agent can read URL and execute."
- UGC is weaker because users see creation as storage, not as content publishing.

Reuses:
- Current PRD URL contract, TTL, delete token, MCP create/get/delete.

### Approach B: Agent URL Schoolhouse

Summary: Homepage is a small set of curated executable lessons, not a prompt library. Each lesson proves the same pattern: give an agent this URL, the agent reads the capsule, then produces a visible result.

Effort: L
Risk: Medium

Pros:
- Best fit for AI 小白.
- Teaches the new behavior by use, not tutorial text.
- Turns Web + MCP + skill into one coherent product surface.
- Creates natural UGC: once users learn the pattern, they package their own repeatable agent tasks.

Cons:
- Seed quality must be high. Bad first capsules break the thesis.
- The design must avoid looking like courseware or a gamified toy.
- Requires two good first-win tracks: general 小红书 and Codex/dev flow.

Reuses:
- All prior `open-first + utility path` decisions.
- Existing visual/interaction demos as feel references, not final UI.
- Existing PRD contract for `/c`, `/view`, `/new`, MCP, CLI/skill.

### Approach C: Curated Agent Content Community

Summary: Homepage is a curated community feed: "today people are using agents to do these things." User capsules are candidates for editor selection.

Effort: XL
Risk: High

Pros:
- Most directly supports the long-term community vision.
- Makes UGC and curation central from day one.
- Has the largest narrative upside if it works.

Cons:
- Too much community surface before literacy is established.
- Governance and editorial load can swamp product work.
- AI 小白 may not understand what to do first.

Reuses:
- Candidate pool and curated pool model.

## Recommended Approach

Choose Approach B: Agent URL Schoolhouse.

It preserves the previous product skeleton but upgrades the north star. The homepage should not say "browse prompts." It should make the user perform the new behavior:

```text
Open capsule -> copy agent instruction -> agent reads URL -> result appears -> make your own capsule.
```

The first release is still a complete product: Web + text endpoint + curated seeds + MCP + skill + launch content.

## Product Contract

### Routes

| Route | Job |
| --- | --- |
| `/` | Teach the agent URL pattern through 2-3 curated capsules. |
| `/new` | Create a capsule without friction. Direct visitors default to `link_only`. |
| `/c/{slug}` | Pure text endpoint for agents and humans. |
| `/view/{slug}` | Human copy page with metadata, copy button, delete path. |
| `/api/capsules` | Create capsule with `discovery_mode`. |
| MCP stdio server | `capsule_create`, `capsule_get`, `capsule_delete`. |
| Skill docs | Teach agents when to create/read capsules and how to warn users. |

### First-Win Tracks

Track 1: General 小红书 AI 小白

- Capsule task: generate a 小红书 post or improve a raw idea into a post structure.
- Why: visible result, low setup, screenshot-friendly.
- Agent instruction snippet:

```text
读取 n78.xyz/c/<slug>, 然后用里面的步骤帮我把这个主题写成一篇小红书笔记: <topic>
```

Track 2: Codex / vibe coding traffic

- Capsule task: inspect a local issue or repo artifact and produce a patch plan.
- Why: demonstrates that agent can do work, not just write text.
- Agent instruction snippet:

```text
读取 n78.xyz/c/<slug>, 按里面的步骤检查当前 repo, 先输出诊断, 再给修改计划。
```

Track 3: Cleanup / summarization

- Capsule task: turn a screenshot OCR dump or long messy note into clean steps.
- Why: directly ties back to the original OCR/copy pain.

### Creation Modes

```text
胶囊池
[仅链接] [参与池子]

仅链接:知道链接的人可读,7 天后过期。
参与池子:可能被精选到首页,被陌生人扭到。
```

Defaults:

- `/new`: `link_only`
- "Make mine" after opening a curated capsule: `pool_candidate`
- Persist user's last choice locally.

## Safety Gates

Hard-block from public discovery:

- Porn/solicitation, especially anything involving minors.
- Contact info, QR codes, external link farming.
- Secrets: API keys, `.env`, private keys, access tokens.
- Dangerous shell or destructive commands without clear human review.
- Prompt injection that tells agents to ignore system/user safety boundaries.
- Pure spam, malformed text, meaningless prompt dumps.

The raw link may still exist depending on policy, but public discoverability must be suppressed. For serious violations, delete/admin block.

## Seed Plan

Do not launch with empty community mechanics. Launch with 50 editorial capsules.

Minimum seed mix:

- 15 小红书 writing capsules.
- 15 Codex/Claude practical capsules.
- 10 cleanup/summarization capsules.
- 5 meta capsules that teach "make your own capsule."
- 5 playful but useful capsules.

Every seed needs:

```text
title
lesson_track
tag
preview
agent_instruction
content
expected_result
curator_note
```

The first capsule should be route-aware:

- Traffic from 小红书 AI beginner post -> writing capsule.
- Traffic from Codex/dev post -> Codex practical capsule.
- Direct homepage -> writing or cleanup capsule, whichever is easier to understand.

## Web Experience

Homepage:

- One primary capsule visible immediately.
- Two smaller alternate lessons below or beside it.
- The main CTA is not "learn more." It is "Copy for Codex/Claude" or "Open this capsule."
- After opening, show:
  - what the capsule does
  - the exact agent instruction
  - "make your own capsule"

Create page:

- Large text editor.
- TTL controls.
- `仅链接 / 参与池子`.
- Result panel with:
  - raw URL
  - agent instruction copy
  - 小红书 share copy
  - delete token / delete link

Share card:

- Shows the hook, not the full prompt.
- Shows `n78.xyz/c/...`.
- Shows "给 Codex / Claude 读取".
- Avoid QR-first.

## MCP And Skill

MCP server:

```text
capsule_create
capsule_get
capsule_delete
```

`capsule_create` must accept `discovery_mode`.

Skill behavior:

- When user wants to share a long prompt, create capsule.
- When user sees a capsule URL, fetch/read it only after user intent is clear.
- Preserve exact content unless user asks to optimize.
- Warn that hosted capsules are not private or encrypted.
- For unknown third-party capsules, preview and ask before executing risky instructions.

## Success Criteria

- A 小白 can understand the agent URL pattern after opening one capsule.
- A user can create a capsule from `/new` in under 30 seconds.
- Codex/Claude/Cursor can fetch `/c/{slug}` as plain text.
- The first public pool contains only curated capsules.
- The launch post can show Web + agent consumption + creation in one loop.
- The MCP server and skill are usable from day one.
- No UI copy implies privacy, encryption, or guaranteed pool exposure.

## What I Noticed

- You rejected "MVP" because the product's moat is polish and taste. That is right for this artifact.
- You pushed the goal beyond prompt storage: "通过一两次使用让用户知道 Agent 还可以这样使用." That is the actual north star.
- You are using 小红书 traffic as a wedge, not as vanity. The product has to be understandable inside a social post.
- You keep separating utility from community. That matters: community should grow from the content object, not be faked with a wall.
